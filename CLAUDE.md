# 営業日報システム

## 概要

営業担当者が日々の活動（顧客訪問・課題・翌日計画）を記録し、上長が承認するシステム。

---

## 機能要件

### 1. 日報 (daily_reports)

- 営業担当者が1日につき1件作成する
- 以下の項目を持つ
  - **報告日** (report_date)
  - **Problem**: 今の課題や相談（テキスト、1件）
  - **Plan**: 明日やること（テキスト、1件）
  - **ステータス**: 下書き / 提出済 / 承認済 / 差し戻し

### 2. 訪問記録 (visit_records)

- 1つの日報に複数件追加可能
- 各訪問記録には以下を記録する
  - **顧客** (customer_id): 顧客マスタから選択
  - **訪問時刻** (visit_time)
  - **訪問目的・内容メモ** (purpose)
  - **担当案件・商品** (case_product)
  - **次回アクション** (next_action)

### 3. コメント (comments)

- Problem・Plan のどちらかを対象に複数件追加可能
- ログイン中の全ユーザー（営業・マネージャー問わず）が投稿可能
- 対象種別 (`target_type`): `problem` / `plan`

### 4. 承認フロー

- 営業が日報を「提出」すると上長（マネージャー）が承認・差し戻しできる
- 承認者と承認日時を日報に記録する

### 5. マスタ

#### 顧客マスタ (customers)

- 会社名、担当者名、電話番号、メールアドレス、住所

#### ユーザーマスタ (users) ※営業マスタ兼マネージャー

- 氏名、メールアドレス、ロール (`sales` / `manager`)

---

## ステータス遷移

```
draft（下書き）
  └─[提出]→ submitted（提出済）
               ├─[承認]→ approved（承認済）
               └─[差し戻し]→ rejected（差し戻し）
                               └─[再提出]→ submitted
```
